# Release v1.4.0

[![GitHub Downloads](https://img.shields.io/github/downloads/alexdelprete/ha-abb-fimer-pvi-vsn-rest/v1.4.0/total?style=for-the-badge)](https://github.com/alexdelprete/ha-abb-fimer-pvi-vsn-rest/releases/tag/v1.4.0)

**Release Date:** 2026-03-25

**Type:** Minor release - Stale-value guard, Sensor Scope metadata,
display name standardization, CI/infra improvements.

## What's Changed

### Added

- **Sensor Scope metadata field** - New "Sensor Scope" column in mapping classifies all 40
  `total_increasing` sensors by reset behavior:
  - `lifetime` (20): energy totals and cycle counters that should never decrease
  - `runtime` (16): energy counters and uptime that reset on reboot
  - `periodic` (4): weekly/monthly energy counters that reset on period boundary
- **Stale-value guard for lifetime sensors** - Lifetime sensors now reject incoming values
  that are lower than the current HA entity state. During inverter startup, the datalogger
  may return 0 or stale values before full internal communication is established. HA's
  `total_increasing` interprets value drops as meter resets, causing massive dashboard spikes.
  Per HA developer docs: "the integration must ensure total_increasing values don't
  erroneously decrease." Thanks to @bryanyork for identifying this issue in PR #42.
- **Display name standardization** - All display names standardized to "Type - Details" pattern
  for optimal grouping in Home Assistant UI.
- **Local brand assets** - Added local brand assets for HA 2026.3 brands proxy API.

### Fixed

- **Display name inconsistencies** - Standardized "Lifetime" suffix across all lifetime
  energy sensors (removed parentheses from 7 sensors: WH, Ein1, Ein2, ETotalApparent,
  PowerPeakAbs, ECt, EDt). ECt and EDt renamed from "Total energy from..." to
  "Energy - Current Transformer (CT) Lifetime" / "Energy - Direct Transducer (DT) Lifetime"
  to follow the established "Energy - ..." naming pattern.

### Infrastructure

- **Devcontainer consolidation** - Removed old `.devcontainer/` folder, unified configuration
  with fixed sudo for pip/uv install, added gh CLI and Claude Code.
- **CI lint enhancements** - Added yamllint and pymarkdown to CI lint workflow.
- **ty type checker** - Added ty type checker to pre-commit config for faster type checking.
- **Coverage flags** - Moved coverage flags from pytest addopts to CI-only, restored
  `--no-cov` in devcontainer pytest args as safety net.
- **strings.json cleanup** - Removed strings.json, using translations/en.json as source of truth.
