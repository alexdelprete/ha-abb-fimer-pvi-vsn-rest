# Release v1.3.6

[![GitHub Downloads](https://img.shields.io/github/downloads/alexdelprete/ha-abb-fimer-pvi-vsn-rest/v1.3.6/total?style=for-the-badge)](https://github.com/alexdelprete/ha-abb-fimer-pvi-vsn-rest/releases/tag/v1.3.6)

**Release Date:** 2026-03-01

**Type:** Minor release - Per-device custom name prefixes for multi-device setups, with
comprehensive bug fixes for `via_device` warnings, entity ID collisions, and discovery
deduplication. Fixes [#36](https://github.com/alexdelprete/ha-abb-fimer-pvi-vsn-rest/issues/36).

Thanks to **@giannicoderani** for reporting [issue #36](https://github.com/alexdelprete/ha-abb-fimer-pvi-vsn-rest/issues/36)
and for extensive beta testing with diagnostic data.

## What's Changed

### Added

- **Per-Device Custom Name Prefixes** - Users with multiple devices of the same type (e.g.,
  2 batteries, 2 inverters) can now set individual custom names for each device via the
  options flow. Single-device setups are unchanged (backward compatible).
  - Multiple devices per type: indexed fields (`prefix_battery_1`, `prefix_battery_2`, etc.)
  - Devices sorted by `device_id` for stable, deterministic ordering
  - Supports up to 4 devices per type
- **`strings.json` as Canonical English Source** - Following Home Assistant convention,
  `strings.json` is now the canonical English source. Translation generator reads from
  `strings.json` and syncs to `translations/en.json`.
- **Collision Safety Net** - Entity ID regeneration and auto-migration now use a two-phase
  approach: pre-compute all renames, detect collisions before executing, skip losers with
  clear warning.
- **Automated `strings.json` Sync** - `convert_to_json.py` now auto-updates sensor entries
  in `strings.json` from mapping data, closing the pipeline automation gap.
- **Debug Logging** - Trace logging for prefix mapping in `build_prefix_by_device()`,
  `_regenerate_entity_ids()`, and options flow for troubleshooting multi-device setups.

### Fixed

- **Inverted Device Prefixes** - Fixed custom name prefixes being assigned to wrong devices
  in multi-device setups (e.g., battery 1 prefix applied to battery 2). Created shared
  `build_prefix_by_device()` helper used consistently across all code paths.
- **`via_device` Deprecation Warnings** - Fixed two separate issues:
  1. Sensor entities were created before the datalogger parent device was registered.
  2. Datalogger device registration used raw `logger_sn` (with colons for VSN700 MACs)
     while `sensor.py` used cleaned `device_id` (without colons) for `via_device`.
- **Entity ID Collisions** - Fixed 3 mapping collision groups where different measurements
  produced identical display names:
  - `VgridL1_N` → "Voltage AC - Grid L1-N" (was "Voltage AC - Phase A-N", conflicted with `PhVphA`)
  - `VgridL2_N` → "Voltage AC - Grid L2-N" (was "Voltage AC - Phase B-N", conflicted with `PhVphB`)
  - `PbaT` → "Power - Battery Aggregate" (was "Power - Battery Total", conflicted with `Pba`)
- **Duplicate Datalogger in Discovery** - Fixed VSN700 datalogger appearing twice in
  discovery (once from status data, once from livedata with empty points array), which
  caused incorrect device counting for prefix key resolution.

### Changed

- **Options Flow UX** - Added step description guiding users to check S/N under each field
  for multi-device setups. Changed indexed field descriptions from `S/N: {serial}` to
  `Sets name for device with S/N: {serial}`.
- **Shared Constants** - Added `TYPE_TO_CONF_PREFIX` mapping in `const.py` for reuse across
  `sensor.py` and `config_flow.py`.
- **Alias Cleanup** - Removed backward-compatible `_compact_serial_number` /
  `_format_device_name` aliases from `sensor.py`; uses direct imports from `helpers.py`.

### Translations

- Full translation coverage for all 10 languages (en, de, es, et, fi, fr, it, nb, pt, sv)
  with per-device indexed prefix strings, S/N placeholders, and improved field descriptions.

## Post-Upgrade Notes

After upgrading from v1.3.5, if you have multiple devices of the same type, you should:

1. Go to Settings > Devices & Services > ABB/FIMER PVI VSN REST
2. Click "Configure"
3. Set custom names for each device — **check the S/N under each field** to match
   it to the correct physical device
4. Check "Regenerate Entity IDs"
5. Click Submit

For single-device-per-type setups, no action is required.
