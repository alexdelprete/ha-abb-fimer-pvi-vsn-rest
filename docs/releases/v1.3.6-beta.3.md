## What's Changed

### Bug Fixes

- **`via_device` Identifier Mismatch** - Fixed datalogger device registration using raw
  `logger_sn` (with colons for VSN700 MACs like `0c:1c:57:fd:c6:2c`) while `sensor.py`
  referenced `via_device` using cleaned `device_id` (without colons: `0c1c57fdc62c`).
  Now both use the cleaned `device_id` from `DiscoveredDevice`, eliminating the HA
  deprecation warning. Thanks @giannicoderani!
- **Device S/N in Options Flow** - Options flow previously showed generic labels like
  "Battery 1", "Battery 2" without serial numbers, making it impossible to identify which
  field maps to which physical device. Now shows `S/N: {serial}` in field descriptions
  via `description_placeholders` so users can correctly assign custom names to the right
  devices. Thanks @giannicoderani!

### Translations

- Updated all 10 language files (en, de, es, et, fi, fr, it, nb, pt, sv) with S/N
  placeholder support in indexed prefix field descriptions

### Tests

- Updated device registry tests to reflect new datalogger discovery-based lookup
