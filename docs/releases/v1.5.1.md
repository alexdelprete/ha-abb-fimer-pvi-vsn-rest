# Release v1.5.1

[![GitHub Downloads](https://img.shields.io/github/downloads/alexdelprete/ha-abb-fimer-pvi-vsn-rest/v1.5.1/total?style=for-the-badge)](https://github.com/alexdelprete/ha-abb-fimer-pvi-vsn-rest/releases/tag/v1.5.1)

**Release Date:** 2026-03-30

**Type:** Patch release - Fixes negative energy values on HA Energy Dashboard. Fixes #43.

## What's Changed

### Fixed

- **Negative solar production on Energy Dashboard** (Fixes #43) - 32 periodic energy sensors
  (DayWH, WeekWH, MonthWH, YearWH, and all `_7D`/`_30D`/`_1Y` series) had
  `state_class: "total"` instead of `"total_increasing"`. Per
  [HA developer docs](https://developers.home-assistant.io/docs/core/entity/sensor/#state-class-sensorstateclasstotal),
  `total` calculates negative deltas when values drop (e.g., 5kWh to 0 = -5kWh), while
  `total_increasing` correctly interprets drops as meter resets. Thanks to @nonth for
  reporting this issue.

### Added

- **Accumulation Mode attribute** - New `accumulation_mode` field in sensor mapping drives
  `state_class` derivation in the generator. Separation of concerns:
  - `accumulation_mode`: determines state_class (`"monotonic"` = total_increasing,
    `"bidirectional"` = total)
  - `sensor_scope`: determines stale-value guard behavior (`"lifetime"`, `"runtime"`,
    `"periodic"`)

### Removed

- **Stale mapping copies** - Removed `docs/vsn-sunspec-point-mapping.json` and
  `docs/vsn-sunspec-point-mapping.xlsx` which were never updated by the regeneration pipeline
  and had diverged from the canonical runtime copy.

### Impact on Existing Users

After upgrading, HA will start treating value drops on periodic energy sensors as meter resets
instead of negative deltas. Existing negative values in long-term statistics won't be
retroactively fixed — users may need to manually correct them via Developer Tools, Statistics.
