# Release Notes - v1.3.6-beta.4

[![GitHub Downloads](https://img.shields.io/github/downloads/alexdelprete/ha-abb-fimer-pvi-vsn-rest/v1.3.6-beta.4/total?style=for-the-badge)](https://github.com/alexdelprete/ha-abb-fimer-pvi-vsn-rest/releases/tag/v1.3.6-beta.4)

**Release Date**: 2026-02-24

**Focus**: Improve options flow UX for multi-device setups and add debug logging for
prefix mapping traceability (#36)

______________________________________________________________________

## What's Changed

### UX Improvements

- **Options Flow Step Description** - Added instructional text at the top of the options
  form: "For devices with multiple units, check the S/N under each field to match it to
  the correct physical device." This guides users to verify serial numbers before entering
  custom names.
- **Clearer Indexed Field Descriptions** - Changed multi-device field descriptions from
  `S/N: {serial}` to `Sets name for device with S/N: {serial}` to make the purpose of
  each field explicit. Applies to all indexed prefix fields (inverter 1-4, meter 1-4,
  battery 1-4).

### Debug Logging

- **Prefix Mapping Trace** - Added debug logging in `build_prefix_by_device()` showing
  each `compact_serial → prefix` mapping, helping trace which custom name maps to which
  physical device.
- **Regeneration Trace** - Added debug log in `_regenerate_entity_ids()` showing the full
  prefix mapping used during entity ID regeneration.
- **Options Flow Trace** - Added debug log showing device S/N placeholders when the
  options form is displayed.

### Translations

- Updated all 10 language files (en, de, es, et, fi, fr, it, nb, pt, sv) with improved
  field descriptions and step description
