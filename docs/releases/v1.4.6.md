# Release v1.4.6

[![GitHub Downloads](https://img.shields.io/github/downloads/alexdelprete/ha-abb-fimer-pvi-vsn-rest/v1.4.6/total?style=for-the-badge)](https://github.com/alexdelprete/ha-abb-fimer-pvi-vsn-rest/releases/tag/v1.4.6)

**Release Date:** TBD

**Type:** Minor feature release - Known devices persistence, partial discovery repair notifications, and automatic re-discovery.

## What's Changed

### Added

- **Known devices persistence** - The integration now remembers all discovered devices across
  restarts in `config_entry.data["known_devices"]`. This enables detection of missing devices
  when the system starts up with partial availability (e.g., datalogger online but inverter
  still powering up at sunrise).

- **Partial discovery repair notification** - When known devices are missing at startup, a
  WARNING-level repair notification appears listing the missing devices with possible causes
  and instructions. The notification clears automatically when all devices recover.

- **Automatic re-discovery** - The coordinator runs `discover_vsn_device()` every poll cycle
  (default: 60s) when devices are missing. When recovered devices are found, the integration
  auto-reloads to create their entities — no manual reload needed.

- **Idempotent device detection** - If the normalized data contains device IDs not present in
  the known devices list (e.g., a new device was connected, or a deleted device is still
  physically present), the integration triggers a reload for automatic device creation.

### Changed

- **Device deletion always allowed** - `async_remove_config_entry_device()` now always returns
  `True`. When a device is deleted via the HA UI, it is removed from the `known_devices` list.
  If the device still physically exists, the next discovery will find it and re-add it. Users
  who want to permanently hide a device should disable it instead of deleting it.

- **Removed automatic orphan cleanup** - The `_cleanup_orphaned_devices()` function has been
  removed. Devices are never auto-deleted from the registry. This prevents the scenario where
  a temporarily offline inverter gets deleted at startup, losing all its entities and history.

### Migration

- **Config entry v7 → v8** - New migration populates the `known_devices` list from the
  existing device registry, so existing users immediately have their expected device list
  without needing to re-discover.
