# Release v1.3.7

[![GitHub Downloads](https://img.shields.io/github/downloads/alexdelprete/ha-abb-fimer-pvi-vsn-rest/v1.3.7/total?style=for-the-badge)](https://github.com/alexdelprete/ha-abb-fimer-pvi-vsn-rest/releases/tag/v1.3.7)

**Release Date:** 2026-03-01

**Type:** Patch release - Fix SysTime timezone double-offset and auto-cleanup orphaned
devices from device registry.

Thanks to **@giannicoderani** for reporting both issues in
[#36](https://github.com/alexdelprete/ha-abb-fimer-pvi-vsn-rest/issues/36).

## What's Changed

### Fixed

- **SysTime Timezone Double-Offset** - The `sys_time` sensor displayed the wrong time
  (e.g., UTC+2 instead of UTC+1 in Italy). The inverter returns local time in Aurora epoch
  seconds, but `datetime.fromtimestamp(..., tz=tz)` interpreted it as UTC and added the
  timezone offset again. Fixed by removing the timezone parameter — the value is already
  local time.
- **Ghost Datalogger Device** - Orphaned devices left behind by earlier discovery bugs
  (e.g., the beta.1 duplicate datalogger) are now automatically cleaned up during
  integration setup. Previously these phantom devices could not be deleted.
- **Device Deletion Protection** - `async_remove_config_entry_device()` now uses smart
  logic: devices still present in discovery are protected from deletion, while orphaned
  devices (no longer discovered) can be removed via the UI.
