# Release v1.5.2

[![GitHub Downloads](https://img.shields.io/github/downloads/alexdelprete/ha-abb-fimer-pvi-vsn-rest/v1.5.2/total?style=for-the-badge)](https://github.com/alexdelprete/ha-abb-fimer-pvi-vsn-rest/releases/tag/v1.5.2)

**Release Date:** 2026-05-20

**Type:** Patch release - Removes the `last_updated` sensor attribute to stop recorder
database growth. Fixes #56.

## What's Changed

### Fixed

- **Recorder database growth from the `last_updated` attribute** (Fixes #56) - Every sensor
  exposed a `last_updated` attribute sourced from the coordinator timestamp, which changed
  on every poll cycle. Home Assistant's recorder deduplicates entity attributes by hashing
  the serialized attributes JSON; because `last_updated` changed every poll while every
  other attribute stayed constant, the recorder could never reuse a `state_attributes` row.
  This stored a near-full copy of each sensor's metadata JSON on every state write. In the
  reporter's setup (66 entities, 60 s polling, ~10 days) this produced ~620,000 distinct
  attribute rows and a 368 MB `state_attributes` table, ~99% of it belonging to this
  integration. The attribute is now removed. Thanks to @fabioquarantini
  ([@fabioquarantini](https://github.com/fabioquarantini)) for the detailed analysis and
  report.
- **Unsatisfiable dependency resolution in CI** - A Dependabot update had pinned
  `aiohttp>=3.13.5` in `pyproject.toml`, which conflicts with the `aiohttp==3.13.3` exact
  pin of the Home Assistant versions installable on Python 3.13. The constraint is
  realigned to `>=3.13.3`, matching `manifest.json` and `requirements.txt`.

### Breaking Changes

- The `last_updated` attribute is no longer present on this integration's sensors.
  Automations or templates reading `state_attr('sensor.x', 'last_updated')` should use the
  core state property instead — Home Assistant already tracks update timing natively via
  `last_changed`, `last_updated`, and `last_reported` on every entity's state object
  (available in templates, the UI, and the API).

### Dependencies

- Update `pip` requirement to `>=26.1.1,<26.2`
- Update dev dependencies: `pytest` (`>=9.0.3`), `pytest-asyncio`, `ruff` (`>=0.15.10`),
  `pytest-homeassistant-custom-component`
- Bump `softprops/action-gh-release` to `3.0.0` and `dependabot/fetch-metadata` to `3`

### Documentation

- Fix pre-existing MD032 markdown violations in older CHANGELOG and README sections

### Impact on Existing Users

After upgrading, the `last_updated` attribute disappears from all sensors and the recorder
can again deduplicate their (now stable) attributes, collapsing to roughly one
`state_attributes` row per entity. Existing oversized attribute history is not retroactively
removed — it is purged over time by the recorder's normal `purge_keep_days` retention, or
can be reclaimed sooner via the `recorder.purge` service.
