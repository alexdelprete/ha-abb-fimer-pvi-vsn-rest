## What's Changed

### Fixed

- **Stale Entity ID Rename Preview (actual fix)** - Fixed HA's "rename device entity IDs"
  feature proposing wrong entity IDs with old category prefixes (e.g., `device_product_number`
  instead of `product_number`). Root cause verified via `.storage/core.entity_registry`:
  the `object_id_base` field retained old prefixed names (e.g., "Device - Product Number")
  while `original_name` was already correct ("Product Number"). HA's rename preview uses
  `object_id_base` with priority over `original_name`. New v6 config entry migration clears
  stale `object_id_base` values so HA falls through to `original_name`. v1.4.3 incorrectly
  targeted `suggested_object_id` which was already `None`.
