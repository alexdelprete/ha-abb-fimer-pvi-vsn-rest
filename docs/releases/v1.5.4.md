# Release v1.5.4

[![GitHub Downloads](https://img.shields.io/github/downloads/alexdelprete/ha-abb-fimer-pvi-vsn-rest/v1.5.4/total?style=for-the-badge)](https://github.com/alexdelprete/ha-abb-fimer-pvi-vsn-rest/releases/tag/v1.5.4)

**Release Date:** 2026-05-31

**Type:** Patch release - Options flow fix.

## What's Changed

### Fixed

- **Recovery Script could not be cleared via the Options flow.** Clearing the Recovery
  Script field and submitting left the old value attached to the config entry, and the
  script continued to run on the next connection failure. The options schema used
  `vol.Optional(CONF_RECOVERY_SCRIPT, default=saved_value)`; when the field is cleared,
  Home Assistant omits the key from the submission and voluptuous refills it with the
  saved default, resurrecting the value. The field now uses
  `description={"suggested_value": ...}`, which pre-fills the form for display without
  resurrecting the value on an empty submission.
- **Per-device name prefixes could not be cleared either.** The custom device-name prefix
  fields in the Options flow had the identical `default=` clear-vs-pre-fill bug, so
  clearing a prefix to revert to default naming did not stick. Fixed with the same
  `description={"suggested_value": ...}` pattern.
