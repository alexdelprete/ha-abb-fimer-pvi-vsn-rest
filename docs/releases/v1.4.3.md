## What's Changed

### Fixed

- **Stale Entity ID Rename Preview** - Fixed HA's "rename device entity IDs" feature proposing
  wrong entity IDs with old category prefixes (e.g., `device_product_number` instead of
  `product_number`). Root cause: HA uses the persisted `suggested_object_id` registry field for
  rename computation, which is only updated when the entity platform loads. For entities in
  "restored" state (device offline), this field retained old values from before the v1.4.1
  display name cleanup. New v5 config entry migration clears stale values so HA falls through
  to the corrected `original_name` field (fixed in v1.4.2).
