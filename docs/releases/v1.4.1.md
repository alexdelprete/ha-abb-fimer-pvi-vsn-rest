# Release v1.4.1

[![GitHub Downloads](https://img.shields.io/github/downloads/alexdelprete/ha-abb-fimer-pvi-vsn-rest/v1.4.1/total?style=for-the-badge)](https://github.com/alexdelprete/ha-abb-fimer-pvi-vsn-rest/releases/tag/v1.4.1)

**Release Date:** 2026-03-25

**Type:** Patch release - Clean sensor names and automatic entity ID migration.

## What's Changed

### Breaking Changes

**Sensor names and entity IDs have changed.** Category prefixes like "Inverter - ", "Device - ",
"System - ", "Battery - ", and "Grid - " have been removed from 57 sensor display names because
the device already provides that context.

For example:

- "Inverter - Configuration Options" → "Configuration Options"
- "Inverter - Type" → "Type"
- "Device - Manufacturer" → "Manufacturer"
- "Device - Serial Number" → "Serial Number"
- "System - Uptime" → "Uptime"
- "Battery - State of Charge" → "State of Charge"
- "Battery - Charge Cycles" → "Charge Cycles"

Some names keep a qualifier to avoid ambiguity:

- "Firmware - Version" → "Firmware Version"
- "Datalogger - Serial Number" → "Logger Serial Number"
- "Datalogger - Board Model" → "Logger Board Model"
- "Datalogger - ID" → "Logger ID"

**Automatic migration:** The integration will automatically rename your existing entity IDs on
the first startup after updating. Your dashboards, automations, and scripts should continue to
work without changes.

**If something doesn't update automatically**, you can manually trigger entity ID regeneration
from Settings → Devices & Services → ABB/FIMER PVI VSN REST → Configure → check
"Regenerate Entity IDs" and submit.

### Fixed

- **Redundant Entity ID Prefixes** - Fixed entity IDs containing redundant category prefixes
  when users set custom device names (e.g., `sensor.abb_fimer_inverter_inverter_configuration_options`
  → `sensor.abb_fimer_inverter_configuration_options`)
- **Wrong Category in Entity IDs** - Fixed datalogger entities incorrectly containing "inverter"
  prefix (e.g., `sensor.abb_fimer_datalogger_inverter_type` → `sensor.abb_fimer_datalogger_type`)

### Changed

- **One-Time Entity ID Migration** - Config entry version bump (1→2) triggers automatic entity ID
  migration on first startup. Runs exactly once and never interferes with user customizations.
- **Removed Old Migration** - Replaced per-startup `_async_migrate_entity_ids` with one-time
  version-gated migration that respects user-customized entity IDs.
- **Removed Dead Code** - Removed non-functional `_attr_suggested_object_id` assignment
  (`suggested_object_id` is not a settable attribute in HA core).
