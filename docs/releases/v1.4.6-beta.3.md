# Release v1.4.6-beta.3

[![GitHub Downloads](https://img.shields.io/github/downloads/alexdelprete/ha-abb-fimer-pvi-vsn-rest/v1.4.6-beta.3/total?style=for-the-badge)](https://github.com/alexdelprete/ha-abb-fimer-pvi-vsn-rest/releases/tag/v1.4.6-beta.3)

**Release Date:** 2026-03-29

**Type:** Beta release - Device sync tightening and code refactoring.

## What's Changed (since beta.2)

### Fixed

- **Client device sync after re-discovery** - After `_attempt_rediscovery()` found recovered
  devices, the coordinator updated its own `discovered_devices` but the client's device list
  and `_device_type_map` remained stale until config reload. Now calls
  `client.update_discovered_devices()` to keep both in sync immediately.

- **Datalogger device_type injection** - The datalogger's livedata key is the MAC address
  (e.g., `a4:06:e9:7f:42:49`) but its discovered device IDs were both the serial number.
  Added `livedata_device_id` field to `DiscoveredDevice` to enable matching, ensuring
  consistent `device_type` injection for all devices.

- **Device type injection logging** - Previously logged every poll cycle (every 15-60s).
  Now logs only once when the injection map is built.

### Refactored

- **client.py** - Extracted `update_discovered_devices()`, `_ensure_device_type_map()`, and
  `_inject_device_types()` helpers. Device type map built lazily on first poll, injection
  is a fast dict lookup.

- **coordinator.py** - Extracted `_update_discovery_state()`, `_sync_known_devices()`,
  `_handle_full_recovery()`, `_handle_partial_recovery()` from `_attempt_rediscovery()`.

- **\_\_init\_\_.py** - Extracted `_sync_known_devices()` and `_detect_missing_devices()`
  from `async_setup_entry()`. Setup flow now reads: sync → detect → notify.
