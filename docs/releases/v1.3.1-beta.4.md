# Release Notes - v1.3.1-beta.4

[![GitHub Downloads](https://img.shields.io/github/downloads/alexdelprete/ha-abb-fimer-pvi-vsn-rest/v1.3.1-beta.4/total?style=for-the-badge)](https://github.com/alexdelprete/ha-abb-fimer-pvi-vsn-rest/releases/tag/v1.3.1-beta.4)

## Overview

This beta release adds startup failure notifications and improves the options dialog UI.

## Changes Since beta.3

### New Features

- **Startup failure notifications** - New option to notify when device is unreachable at Home Assistant startup/restart
- **Separate runtime vs startup notifications** - Users can now enable/disable each type independently

### Configuration Changes

- **Renamed option**: "Enable connection failure notifications" â†’ "Enable runtime failure notifications"
- **New option**: "Enable startup failure notifications" (default: **disabled**)
- **Compact UI**: All option descriptions shortened to single lines to reduce scrolling

### Default Values

| Option | Default |
|--------|---------|
| Enable runtime failure notifications | On |
| Enable startup failure notifications | **Off** |
| Failures before notification | 3 |
| Recovery script | (empty) |

**Note**: Startup notifications are disabled by default because most users have solar-only systems where the inverter is offline at night. Users with battery systems who want startup notifications can enable it in the options dialog.

- **README updated** with comprehensive Configuration Options section
