## Overview

This release updates dependencies to address security vulnerabilities identified by Dependabot and standardizes development tooling versions.

______________________________________________________________________

## Breaking Changes

### Home Assistant Minimum Version: 2026.1.0

**This release requires Home Assistant 2026.1.0 or newer.**

The minimum Home Assistant version has been updated to 2026.1.0 because it is the first version that includes the security-patched `aiohttp>=3.13.3`.

**Impact**: Users running Home Assistant versions older than 2026.1.0 will not be able to install or update to this release.

**Your options:**

1. **Upgrade Home Assistant** to 2026.1.0 or newer, then update this integration
2. **Stay on v1.3.1** if you cannot or do not want to upgrade Home Assistant yet

______________________________________________________________________

## Security Fixes

### aiohttp Updated to 3.13.3

Updated `aiohttp` requirement to `>=3.13.3` to address security vulnerabilities identified by Dependabot.

- **Previous**: `aiohttp>=3.9.0`
- **New**: `aiohttp>=3.13.3`

This security update is automatically satisfied when running Home Assistant 2026.1.0+.

Updated the Ruff linter to version 0.14.11 across all configuration files for consistency:

## Upgrade Notes

### ⚠️ Before You Upgrade

**This is a breaking change.** Ensure you are running Home Assistant 2026.1.0 or newer before upgrading this integration.

**To check your Home Assistant version:**

1. Go to **Settings** → **About**
2. Verify your version is **2026.1.0 or higher**

### If You Are on an Older HA Version

**DO NOT upgrade this integration** if you are running Home Assistant older than 2026.1.0.

- The integration **will not function** on older HA versions
- Stay on **v1.3.1** until you upgrade Home Assistant
- There is no loss of functionality by staying on v1.3.1 (only security/maintenance updates in v1.3.2)

### Recommended Upgrade Path

1. First, upgrade Home Assistant to 2026.1.0 or newer
2. Verify Home Assistant is working correctly
3. Then update this integration to v1.3.2
