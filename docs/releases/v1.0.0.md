## Features

### Device Support

**VSN Dataloggers**:
- VSN300 (WIFI LOGGER CARD) with HTTP Digest authentication
- VSN700 (Ethernet/WiFi) with HTTP Basic authentication

**Inverters** (via datalogger):
- Single-phase models (REACT2-3.6-TL, REACT2-5.0-TL, etc.)
- Three-phase models (PVI-10.0-OUTD, PVI-12.5-OUTD, etc.)

- Energy meters
- Battery storage systems

- **Automatic Discovery**: VSN model and device detection
- **SunSpec Normalization**: 258 data points mapped
- **Aurora Protocol**: 195 operational states translated
- **Multi-Language**: English and Italian
- **Configuration UI**: Easy setup through HA UI
- **Device Hierarchy**: Proper relationships (devices via datalogger)
- **Complete Metadata**: Model, manufacturer, firmware, serial numbers
- **Rich Attributes**: 14+ attributes per sensor for debugging

## Configuration

**Prerequisites**:
- VSN300 or VSN700 datalogger on your network
- Network access to datalogger (HTTP port 80)
- Valid authentication credentials

**Setup Steps**:
1. Go to Settings ‚Üí Devices & Services
2. Click "Add Integration"
3. Search for "ABB FIMER PVI VSN REST"
4. Enter configuration:
   - **Host**: IP address or hostname of VSN datalogger
   - **Username**: Default is "guest"
   - **Password**: Default is empty (leave blank)
   - **Scan Interval**: Default 60 seconds (30-600 range)
5. Click Submit

The integration will:
- Auto-detect VSN model (VSN300 or VSN700)
- Discover all connected devices
- Create entities for all available data points

## Migration Guide

### From Beta Versions

If you're upgrading from any beta version:

1. **Backup** your Home Assistant configuration
2. **Update** via HACS or manual installation
3. **Restart** Home Assistant
4. **Verify** all entities are working

**Note**: Entity IDs are stable since beta.26. No entity ID changes in v1.0.0.

### From Legacy Integration (ha-abb-powerone-pvi-sunspec v4.x)

If you're migrating from the legacy Modbus integration:

‚ö†Ô∏è **Important**: This integration uses a **different protocol** (REST API via VSN datalogger) than the legacy integration (Modbus/TCP direct). You need a VSN300 or VSN700 datalogger to use this integration.

**For Direct Modbus Access**: Use [ha-abb-fimer-pvi-sunspec](https://github.com/alexdelprete/ha-abb-fimer-pvi-sunspec) instead.

**For VSN Datalogger Users**:
1. Install new integration alongside legacy
2. Verify all devices discovered
3. Update dashboards/automations to new entity IDs
4. Test thoroughly
5. Remove legacy integration when satisfied

## Known Limitations

1. **Requires VSN Datalogger**: This integration only works with VSN300 or VSN700 dataloggers
2. **Polling-Based Updates**: Data fetched via polling (default 60s, minimum 30s)
3. **Network Dependency**: Requires continuous network access to datalogger
4. **VSN API Limitations**: Some advanced features may not be exposed via VSN REST API
