**Type:** Patch release - Fixes object_id_base migration from v1.4.4.

## What's Changed

### Fixed

- **Fix object_id_base migration** - v1.4.4 incorrectly cleared `object_id_base` to `None`,
  causing HA's rename preview to generate entity IDs with only the device name
  (e.g., `sensor.abb_fimer_datalogger` instead of `sensor.abb_fimer_datalogger_product_number`).
  New v7 migration sets `object_id_base` to `original_name` (the correct value, already fixed
  in v1.4.2's v4 migration).
